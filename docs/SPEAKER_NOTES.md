## 0:07 – 0:12 · §2 GLM-5.1 模型解构

- **屏幕**：`#ch2`，重点讲 MoE SVG。
- **要点**：
  - GLM 家族演进：130B Dense → 4.5/4.6 MoE → 5.1 推断 1T。

  - MoE = 稀疏的"专家委员会"，激活只有 6.4 % 但容量 16×。
  - 训练分四阶段：Pretrain 80 % / Mid 10 % / Post 8 % / Align 2 %。

## 0:12 – 0:18 · §3 训练算力推导

- **屏幕**：`#ch3`，重点滚到滑块。
- **要点**：
  - 写公式 $C \approx 6 N D$，再加 1.33× 工程系数变成 $C_{\text{eff}} = 8 N_{\text{act}} D$。
  - 代入：8 × 6.4×10¹⁰ × 2×10¹³ ≈ **1.024 × 10²⁵ FLOPs**。提醒学生这个数字接近美国 EO 14110 的 10²⁶ 报备阈值。
- **互动**：现场拨滑块，让学生回答"如果 BISTU 申请到 1024 张 B200，要训练几天？" — 答案约 184 天。
- 再切到曲线图说明：H100 → H200 → B200 → GB200 之间训练时长差距。

## 0:18 – 0:24 · §4 NVIDIA GPU 选型

- **屏幕**：`#ch4`
- **要点**：
  - 四张卡比较卡片：BF16 / HBM / NVLink / TDP / 价格。
  - 横向柱状图：B200 BF16 算力是 H100 的 2.27×。
  - 选型矩阵：5 种使用场景给出推荐。
  - **本课选型**：2 048 × B200 + IB-XDR 800G 主集群；可扩展到 32 个 GB200 NVL72 机柜。

## 0:24 – 0:31 · §5 集群架构

- **屏幕**：`#ch5`
- **要点**：
  - 单节点 8 GPU + 8 NIC + 2 CPU + 2 TB DDR5 + 30 TB NVMe，14.3 kW。
  - 节点内 NVSwitch 拓扑 SVG → 强调 "NVLink 域 ≤ 8 是 Hopper 时代，NVL72 时代变成 ≤ 72"。
  - 机柜密度：风冷 32 GPU/柜，液冷 64 GPU/柜，NVL72 整柜 72 GPU。
  - IB Fat-tree 三层：64 leaf + 32 spine + 16 core = 112 台交换机 ≈ 1.7 亿。
  - Spectrum-X 以太网作为 CAPEX -15-20% 的备选。

## 0:31 – 0:37 · §6 并行策略

- **屏幕**：`#ch6`
- **要点**：
  - 六维并行表 (DP / TP / PP / EP / SP / CP) — 这是同学最容易混的。
  - ZeRO 三阶段表：解释为什么 B200 时代 ZeRO-1 就够。
  - **推荐配置 TP=8, PP=8, EP=8, DP=32, CP=1**。
  - 显存饼图：B200 192GB 用了 17 GB 权重 + 余量做激活。
  - 框架选型：Megatron-Core + Transformer Engine FP8 + FlashAttention-3。

## 0:37 – 0:41 · §7 存储与数据流水线

- **屏幕**：`#ch7`
- **要点**：

  - 三层存储推荐：5 PB 热 / 50 PB 温 / 100 PB 冷。

  - 数据预处理 5 步流水。

  - 框架对比表：vLLM / TensorRT-LLM / NVIDIA Dynamo / Triton / SGLang。

## 0:51 – 0:56 · §10 TCO

- **屏幕**：`#ch10`
- **要点**：
  - Baseline KPI：CAPEX ~17 亿、3 年 OPEX ~3.6 亿、TCO ~20.8 亿。
  - 五种硬件 × 网络组合堆叠柱状图。

## 课堂常见问题速答

> 给老师备查。

**Q1：为什么 H100 还在卖却已经"过时"？**
A：Blackwell 训练性价比是 Hopper 的 2.3-2.7×，但 H100 仍有：① 库存货稳定不受新一批出口管制波及；② 软件栈最成熟；③ 推理用 H100 反而比 B200 性价比更高（推理是 HBM 带宽制约，B200 的 8TB/s 不一定全用上）。

**Q2：MFU 36 % 是不是太低？**
A：实际工业值。GPT-4 训练披露 32-40 %，DeepSeek-V3 报告 ~40 %（H800），MLPerf v4.1 顶尖结果 50-55 % 但都是优化过的小模型。1T MoE 在 2 048 卡规模上 36-42 % 已经是工程奇迹。

**Q3：为什么不选 Google TPU？**
A：(1) TPU 不外卖，只能在 GCP 用；(2) 国内合规与网络成本高；(3) 软件栈（Pax / JAX / XLA）国内人才稀缺。NVIDIA + PyTorch 是国内事实标准。

**Q4：国产卡（华为 Ascend 910C / 寒武纪 / 摩尔线程）能不能替代？**
A：单卡算力 910C 已接近 H100 80 %；但软件栈（CANN / MindSpore）、互联（HCCS 384 GB/s vs NVLink 1800 GB/s）、生态（社区代码、第三方框架）仍需 1-2 年追赶。本课作为 B 计划必须考虑，不可作为主方案。

**Q5：为什么是 PUE 1.15 而不是 1.08（浸没）？**
A：综合考虑：板式液冷 (DLC) 已经是主流方案，运维成熟，CAPEX 适中；浸没式 PUE 更低但 CAPEX 高 1.5-2×、维护门槛高、占地大。北京气候下 DLC + 自然冷源已能稳定 PUE 1.15。

**Q6：训练完成后这些 GPU 干什么？**
A：转推理！这正是为什么我们在 §9 设计了 256 GPU 推理集群。模型迭代后，老一代 GPU 自然下转推理 / 数据预处理 / 离线评测。GPU 7-8 年生命周期，不会闲置。
